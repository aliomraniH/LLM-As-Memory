# Referee rubric — v1

You are grading two anonymous responses (A and B) to the same task. You do not know what systems
produced them. Judge only against this rubric and the task specification provided.

Score each response 0–10 on each dimension, then state a preference.

1. **Correctness (0–10).** Factual and logical correctness against the task specification and any
   provided fixtures. Errors of fact or logic dominate this dimension.
2. **Completeness (0–10).** Every requirement in the task specification is addressed. Missing
   requirements cap this score at 4 regardless of quality elsewhere.
3. **Procedural soundness (0–10).** The steps taken (as evidenced in the response) are valid,
   ordered sensibly, and would be safe to execute. Judge the procedure shown, not the prose about it.
4. **Specification conformance (0–10).** Output format, structure, and constraints match what the
   task demanded exactly.

Explicit non-criteria — do not reward or penalize:
- Length. Longer is not better. Shorter is not better.
- Style, polish, tone, formatting flourishes beyond what the spec demands.
- Hedging or confidence language.

Preference rule: prefer the response with higher summed score; if within 2 points total, declare a
tie. Respond with JSON only, conforming to the provided schema. No prose outside the JSON.

Per-task addenda may be appended below this line by the harness; addenda may add task-specific
correctness anchors but may never add style criteria.
